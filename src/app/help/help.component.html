Overview:

BoneTwin is an analytical tool designed to facilitate the pair-matching of commingled skeletal elements by providing the probability that a pair of bones originate from the same individual or from different individuals, along with key supporting statistics. Prior to using BoneTwin, users must first superimpose 3D mesh models of the bones and calculate the mean distance (Dmean) and standard deviation of distances (Dsd) between corresponding faces using CloudCompare (https://cloudcompare.org/index.html). These variables (Dmean and Dsd) serve as inputs for the logistic regression models in BoneTwin.

How to use BoneTwin:

  1. Population
    • Choose the reference population from which the bones are most likely derived.
    • 'Korean' represents a modern South Korean population.
    • 'American (modern)' represents a modern U.S. White population.
    • 'American (WWII)' represents U.S. servicemen who died during WWII.
  2. Sex
    • Select the biological sex of the bones being compared.
    • BoneTwin does not support 'pooled sex' or 'unknown sex' categories. If sex is unknown, conduct two separate analyses (one as male and one as female) and report both sets of results.
    • For the 'American (WWII)' population, only the 'Male' option is available.
  3. Bone
    • Choose the type of skeletal element to be compared.
    • Clavicles are available only for the 'American (modern)' population dataset.
  4. ROC Graph and AUC
    • BoneTwin provides ROC curves and AUC values compare the performance of three logistic regression models: a model using Dmean only, a model using Dsd only, and a model using both Dmean and Dsd.
    • As a general guideline, the model with the highest AUC is recommended, as it indicates superior discriminative performance.
  5. Variable(s)
    • Select which of the three models you wish to use: a model using Dmean only, a model using Dsd only, and a model using both Dmean and Dsd.
  6. Enter values
    • Input the Dmean and/or Dsd values obtained from the CloudCompare analysis.
  7. Output

    BoneTwin does not render a definitive conclusion regarding whether two bones belong to the same individual. Instead, it provides statistically informed outputs to guide expert interpretation.

    ✔ Calculated probability

    • This is the estimated probability that the two bones represent a 'Match' (i.e., originate from the same individual).
    • The probability of a 'Non-match' (i.e., originate from different individuals) can be calculated by subtracting the calculated probability from 1.
    • These probabilities are generated using logistic regression models based on the selected variables.

    ✔ Sensitivity

    • Sensitivity, also known as the 'true positive rate', is the proportion of actual 'Matches' that are correctly identified as 'Matches'.

    ✔ False Positive Rate

    • False positive rate, also expressed as '1 - specificity', is the proportion of actual 'Non-matches' that are incorrectly classified as 'Matches'.
    • Even when the calculated probability is < 0.5, a false positive rate of < 0.05 may still support the conclusion that the bones originate from the same individual.
    • If the calculated probability is > 0.5 but the false positive rate is also > 0.05, caution is warranted before concluding the bones originate from the same individual.
